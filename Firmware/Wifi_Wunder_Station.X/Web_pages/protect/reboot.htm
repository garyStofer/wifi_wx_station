Note on Windows NETBIOS name caching

Windows caches the IP address associated with the NETBIOS names. If the device has changed it's IP address during the reboot a Windows computer will not connect to the board via its name until the cache has been refreshed. The brute force method to reload the cache is to reboot the Windows computer. A slightly better option is to run the following command from a cmd window: nbtstat -R.

Depending on the browser used, you might also need to clear the browser history after clearing the windows cache.
